## Decisions deliberately deferred, not made here

Per this module's design (report, don't silently fix — see the docstring
in `validation.py`): the `weight`/`height`/`foot` sentinel warnings on
`players` are **not** normalized by this check suite. None of the feature
families planned for the spike (`docs/feature-definitions.md`, SLS-013)
are anthropometric, so this is not currently load-bearing — but it stays
visible here rather than being silently resolved, in case that changes.

## Test coverage

18 unit tests in
[`tests/data/test_validation.py`](../tests/data/test_validation.py) cover
every check function — including the foreign-key checks, with a case that
confirms the `sentinel_values` exclusion (e.g. `playerId == 0`) doesn't
mask a real orphan — against small synthetic Polars frames, so the checks
themselves are trustworthy before being pointed at 3.25M real rows.
